📁 Souborové systémy a správa souborů

Téma 23 - Operační systémy

⚙️ Operační systémy 📚 Maturitní okruh

📖 Co je souborový systém?

Souborový systém (file system) je způsob, jakým operační systém organizuje a ukládá data na úložné médium (HDD, SSD, USB). Definuje:

  • Jak jsou data fyzicky uložena na disku
  • Jak jsou soubory pojmenovány a organizovány do složek
  • Kdo má k souborům jaká oprávnění
  • Metadata souborů (velikost, datum vytvoření, vlastník...)

Typy souborových systémů

FAT32 (File Allocation Table)

  • Kompatibilita: Funguje na všech OS (Windows, Mac, Linux)
  • Omezení: Max velikost souboru 4 GB, max velikost oddílu 2 TB
  • Použití: USB flash disky, SD karty, starší zařízení
  • Nevýhody: Žádná oprávnění, žádná šifrování, fragmentace

NTFS (New Technology File System)

  • Výchozí pro Windows (od Windows NT/2000)
  • Max soubor: 16 EB (prakticky neomezeno)
  • Funkce:
    • Žurnálování (journal) – obnova po pádu
    • Oprávnění a šifrování (EFS)
    • Komprese souborů
    • Kvóty – limit místa pro uživatele
  • Nevýhoda: Linux čte nativně, zápis přes driver (ntfs-3g)

ext4 (Fourth Extended File System)

  • Výchozí pro Linux (Ubuntu, Debian...)
  • Max soubor: 16 TB, max oddíl: 1 EB
  • Funkce: Žurnálování, rychlé, spolehlivé
  • Nevýhoda: Windows ho nativně nepodporuje

APFS (Apple File System)

  • Výchozí pro macOS (od High Sierra) a iOS
  • Optimalizován pro SSD
  • Šifrování, snapshotování, klonování souborů

Porovnání souborových systémů

Vlastnost FAT32 NTFS ext4
Max velikost souboru 4 GB 16 EB 16 TB
Oprávnění ❌ ✓ ✓
Žurnálování ❌ ✓ ✓
Kompatibilita Všechny OS Windows, Linux (čtení) Linux

Adresářová struktura

Windows

C:\                          # Kořen systémového disku
├── Windows\                 # Systémové soubory OS
│   ├── System32\           # Jádro systému, DLL knihovny
│   └── Fonts\              # Nainstalovaná písma
├── Program Files\           # 64-bit aplikace
├── Program Files (x86)\     # 32-bit aplikace
├── Users\                   # Uživatelské profily
│   ├── Jan\
│   │   ├── Desktop\        # Plocha uživatele
│   │   ├── Documents\      # Dokumenty
│   │   ├── Downloads\      # Stažené soubory
│   │   └── AppData\        # Nastavení aplikací (skrytá)
│   └── Public\             # Sdílené mezi uživateli
└── ProgramData\            # Data aplikací pro všechny uživatele

Linux (Filesystem Hierarchy Standard)

/                           # Kořen (root) - vše začíná zde
├── bin/                    # Základní příkazy (ls, cp, cat)
├── boot/                   # Bootloader, jádro
├── dev/                    # Zařízení jako soubory (sda, tty)
├── etc/                    # Konfigurační soubory
├── home/                   # Domovské adresáře uživatelů
│   └── jan/               # /home/jan je domov uživatele jan
│       ├── Documents/
│       └── .bashrc        # Skrytý konfigurační soubor
├── lib/                    # Systémové knihovny
├── media/                  # Automaticky připojená média (USB)
├── mnt/                    # Ručně připojená úložiště
├── opt/                    # Volitelný software třetích stran
├── proc/                   # Virtuální – info o procesech
├── root/                   # Domov uživatele root
├── tmp/                    # Dočasné soubory
├── usr/                    # Uživatelské programy
│   ├── bin/               # Běžné příkazy
│   └── share/             # Sdílená data (dokumentace)
└── var/                    # Proměnná data (logy, cache)
    └── log/               # Systémové logy

💡 V Linuxu je vše soubor: I zařízení (disk, klávesnice) jsou reprezentována jako soubory v /dev/.

Příkazy pro správu souborů

Windows (CMD / PowerShell)

# Navigace
cd C:\Users\Jan\Documents   # Změna adresáře
cd ..                       # O úroveň výš
dir                        # Výpis obsahu (ls v PowerShell)

# Práce se soubory
copy soubor.txt kopie.txt   # Kopírování
move soubor.txt ../         # Přesun
del soubor.txt              # Smazání souboru
rename stary.txt novy.txt   # Přejmenování

# Práce s adresáři
mkdir NovaSlozka            # Vytvoření složky
rmdir SlozKa                # Smazání prázdné složky
rmdir /S SlozKa             # Smazání i s obsahem

# Zobrazení obsahu
type soubor.txt             # Zobrazí obsah souboru

# Hledání
dir /S *.txt                # Hledá .txt rekurzivně
findstr "hledany_text" soubor.txt

Linux (Bash)

# Navigace
cd /home/jan/Documents      # Absolutní cesta
cd ~/Documents              # ~ = domovský adresář
cd ..                       # O úroveň výš
pwd                        # Zobrazí aktuální adresář

# Výpis
ls                         # Základní výpis
ls -l                      # Detailní (oprávnění, velikost, datum)
ls -la                     # Včetně skrytých souborů (tečka na začátku)
ls -lh                     # S čitelnými velikostmi (KB, MB)

# Práce se soubory
cp soubor.txt kopie.txt    # Kopírování
cp -r slozka/ nova_slozka/ # Rekurzivní kopírování složky
mv soubor.txt ../          # Přesun
mv stary.txt novy.txt      # Přejmenování (= přesun na stejné místo)
rm soubor.txt              # Smazání
rm -r slozka/              # Smazání složky s obsahem
rm -rf slozka/             # Force bez potvrzení (POZOR!)

# Práce s adresáři
mkdir nova_slozka          # Vytvoření
mkdir -p a/b/c             # Vytvoří i nadřazené

# Zobrazení obsahu
cat soubor.txt             # Celý soubor
head -n 10 soubor.txt      # Prvních 10 řádků
tail -n 10 soubor.txt      # Posledních 10 řádků
less soubor.txt            # Interaktivní prohlížení

# Hledání
find /home -name "*.txt"   # Hledá podle jména
grep "text" soubor.txt     # Hledá v obsahu souboru
grep -r "text" ./          # Rekurzivně v aktuálním adresáři

Oprávnění souborů

Linux (chmod)

Každý soubor má oprávnění pro vlastníka (owner), skupinu (group) a ostatní (others).

ls -l soubor.txt
# -rw-r--r-- 1 jan users 1024 Dec 13 20:00 soubor.txt
#  ↑↑↑↑↑↑↑↑↑
#  │├─┤├─┤├─┤
#  │ │  │  └── others (ostatní): r-- = jen čtení
#  │ │  └───── group (skupina): r-- = jen čtení  
#  │ └──────── owner (vlastník): rw- = čtení + zápis
#  └────────── typ: - = soubor, d = adresář

# Oprávnění:
# r (read)    = čtení        = 4
# w (write)   = zápis        = 2
# x (execute) = spouštění    = 1

# Změna oprávnění
chmod 755 skript.sh        # rwxr-xr-x (7=rwx, 5=r-x)
chmod u+x skript.sh        # Přidá execute pro vlastníka
chmod go-w soubor.txt      # Odebere zápis pro group a others

# Změna vlastníka
chown jan soubor.txt       # Změna vlastníka
chown jan:users soubor.txt # Změna vlastníka i skupiny

Windows (NTFS ACL)

# Zobrazení oprávnění
icacls soubor.txt

# Přidání oprávnění
icacls soubor.txt /grant Jan:F      # F = Full control
icacls soubor.txt /grant Users:R    # R = Read only

# GUI: Pravý klik → Vlastnosti → Zabezpečení

🎓 Shrnutí pro maturitu

  • FAT32: Univerzální kompatibilita, max 4 GB soubor
  • NTFS: Windows výchozí, oprávnění, žurnálování
  • ext4: Linux výchozí, spolehlivý
  • Windows: Vše pod C:\, složky Users, Program Files
  • Linux: Vše pod /, domov v /home/username
  • Oprávnění Linux: rwx pro owner/group/others (chmod 755)
  • Základní příkazy: cd, ls/dir, cp/copy, mv/move, rm/del, mkdir
