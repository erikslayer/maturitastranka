Téma 24 - Operační systémy

⚙️ Operační systémy 📚 Maturitní okruh

📖 Nastavení síťového prostředí

Aby se počítač mohl připojit k síti a internetu, potřebuje správnou síťovou konfiguraci. Klíčové parametry:

  • IP adresa: Unikátní identifikátor zařízení v síti (např. 192.168.1.100)
  • Maska podsítě: Určuje, která část IP je síť a která zařízení (např. 255.255.255.0)
  • Výchozí brána (Gateway): IP routeru, přes který se dostaneme do internetu
  • DNS server: Překládá doménová jména na IP adresy

Statická vs Dynamická IP

Vlastnost Statická IP Dynamická IP (DHCP)
Nastavení Ruční Automatické od DHCP serveru
Změna Nemění se Může se měnit při obnově
Použití Servery, tiskárny Běžné počítače, mobily

🔍 Diagnostické nástroje

ipconfig / ip addr – Zobrazení síťové konfigurace

Základní nástroj pro zjištění aktuálního nastavení síťového adaptéru.

# WINDOWS
ipconfig                  # Základní info (IP, maska, brána)
ipconfig /all            # Detailní info včetně MAC, DHCP, DNS

# Ukázkový výstup:
# Ethernet adapter LAN:
#    IPv4 Address. . . . : 192.168.1.100
#    Subnet Mask . . . . : 255.255.255.0
#    Default Gateway . . : 192.168.1.1

ipconfig /release        # Uvolní DHCP přidělenou adresu
ipconfig /renew          # Požádá DHCP o novou adresu
ipconfig /flushdns       # Vymaže DNS cache (užitečné při problémech s překlady)

# LINUX
ip addr                  # Zobrazí IP adresy všech adaptérů
ip addr show eth0        # Jen konkrétní adaptér
ifconfig                 # Starší verze (deprecated)

ping – Test dostupnosti

Posílá ICMP Echo pakety a měří odezvu. Základní test „funguje síť?".

ping google.com          # Test dostupnosti internetu
ping 192.168.1.1         # Test dostupnosti routeru

# Ukázkový výstup:
# Reply from 142.250.185.206: bytes=32 time=15ms TTL=118
# Reply from 142.250.185.206: bytes=32 time=14ms TTL=118
# 
# Packets: Sent = 4, Received = 4, Lost = 0 (0% loss)
# Minimum = 14ms, Maximum = 15ms, Average = 14ms

# Co znamená výstup:
# - time = latence (menší = lepší, <50ms je OK)
# - TTL = Time To Live (kolik skoků zbývá)
# - Lost = ztráta paketů (mělo by být 0%)

ping -t google.com       # Windows: nekonečný ping (Ctrl+C ukončí)
ping -c 5 google.com     # Linux: jen 5 paketů

💡 Diagnostický postup:

  1. ping 127.0.0.1 – test TCP/IP stacku (localhost)
  2. ping vlastní_IP – test síťové karty
  3. ping výchozí_brána – test připojení k routeru
  4. ping 8.8.8.8 – test připojení k internetu (Google DNS)
  5. ping google.com – test DNS překladu

tracert / traceroute – Sledování cesty

Ukazuje všechny routery na cestě k cíli. Užitečné pro nalezení, kde vzniká problém. 

# Windows
tracert google.com

# Linux
traceroute google.com

# Ukázkový výstup:
#  1     1 ms   192.168.1.1      [router doma]
#  2    10 ms   10.0.0.1         [router poskytovatele]
#  3    15 ms   85.70.62.1       [páteřní síť]
#  4    20 ms   142.250.185.206  [cíl - Google]

# * * * = router neodpovídá nebo blokuje ICMP

netstat – Síťová připojení a porty

Zobrazuje aktivní síťová spojení a naslouchající porty.

netstat -an              # Všechna připojení s čísly portů

# Ukázkový výstup:
# Proto  Local Address         Foreign Address       State
# TCP    0.0.0.0:80           0.0.0.0:0             LISTENING
# TCP    192.168.1.100:52341  142.250.185.206:443   ESTABLISHED

# Stavy:
# LISTENING    = čeká na připojení (server)
# ESTABLISHED  = aktivní spojení
# TIME_WAIT    = spojení se zavírá
# CLOSE_WAIT   = čeká na uzavření

netstat -b               # Windows: zobrazí, který program používá port
netstat -tulpn           # Linux: TCP/UDP s programy

nslookup / dig – DNS dotazy

Dotazuje se DNS serverů na překlad doménových jmen.

nslookup google.com

# Ukázkový výstup:
# Server:  router.local (192.168.1.1)
# 
# Name:    google.com
# Addresses:  142.250.185.206

nslookup google.com 8.8.8.8   # Dotaz na konkrétní DNS server

# Linux: pokročilejší alternativa
dig google.com
dig google.com +short         # Jen IP adresa

arp – ARP tabulka

Zobrazuje mapování IP adres na MAC adresy v lokální síti.

arp -a

# Ukázkový výstup:
# Interface: 192.168.1.100
#   Internet Address      Physical Address      Type
#   192.168.1.1           aa-bb-cc-dd-ee-ff     dynamic
#   192.168.1.50          11-22-33-44-55-66     dynamic

⚙️ Síťové služby

DHCP (Dynamic Host Configuration Protocol)

Automatické přidělování IP adres zařízením v síti.

  • Klient pošle broadcast „Potřebuji IP"
  • DHCP server nabídne IP adresu
  • Klient přijme a potvrdí
  • Přiděluje: IP, masku, bránu, DNS, dobu platnosti (lease)

💡 Lease time: Jak dlouho je IP přidělena. Po vypršení musí klient požádat o prodloužení.

DNS (Domain Name System)

Překlad doménových jmen na IP adresy – internet by bez DNS fungoval jen s IP adresami.

google.com  →  DNS Server  →  142.250.185.206

Veřejné DNS servery:

  • Google: 8.8.8.8, 8.8.4.4
  • Cloudflare: 1.1.1.1, 1.0.0.1 (rychlý, soukromí)
  • OpenDNS: 208.67.222.222

Firewall

Filtruje síťový provoz podle pravidel. Chrání počítač před neoprávněným přístupem. 

# WINDOWS
# GUI: Windows Defender Firewall
netsh advfirewall show allprofiles    # Stav firewallu

# LINUX
sudo ufw status                       # Ubuntu (ufw)
sudo ufw enable                       # Zapnutí
sudo ufw allow 22                     # Povolení SSH portu
sudo ufw deny 80                      # Blokování HTTP

sudo iptables -L                      # Nízkoúrovňová pravidla

🛠️ Řešení běžných problémů

Symptom Možná příčina Řešení
Nejde ping na bránu Kabel, Wi-Fi, DHCP Zkontroluj připojení, ipconfig /renew
Jde ping na IP, ne na doménu DNS problém ipconfig /flushdns, změň DNS
Pomalé připojení Vytížení sítě, špatný signál Speedtest, zkontroluj signál Wi-Fi
IP 169.254.x.x DHCP selhal (APIPA) Zkontroluj DHCP server, restart

🎓 Shrnutí pro maturitu

  • ipconfig/ip addr: Zjištění IP, masky, brány
  • ping: Test dostupnosti, měření latence
  • tracert/traceroute: Cesta paketů – kde je problém
  • netstat: Aktivní připojení a porty
  • nslookup: Test DNS překladu
  • DHCP: Automatické přidělení IP
  • DNS: Překlad doménových jmen (8.8.8.8)
  • Firewall: Blokuje/povoluje síťový provoz
