⚙️ Operační systémy 📚 Maturitní okruh

📖 Správa uživatelů

Moderní operační systémy jsou víceuživatelské – mohou s nimi pracovat různí lidé, každý se svým účtem, nastavením a oprávněními. Správa uživatelů zajišťuje bezpečnost a oddělení dat.

Typy uživatelských účtů

Typ účtu Popis Příklady
Administrátor/root Plná práva – může měnit systémová nastavení, instalovat software, spravovat ostatní uživatele Administrator (Win), root (Linux)
Standardní uživatel Omezená práva – může spouštět programy, upravovat vlastní soubory, nemůže měnit systém Běžné uživatelské účty
Host (Guest) Dočasný přístup s minimálními právy, často bez možnosti ukládání Guest účet
Systémový účet Pro běh služeb a procesů, ne pro přímé přihlášení uživatele SYSTEM, LocalService, daemon

Skupiny uživatelů

Skupiny umožňují hromadné přidělování oprávnění. Místo nastavování práv každému uživateli zvlášť přidáme uživatele do skupiny a oprávnění má celá skupina.

  • Windows: Administrators, Users, Power Users, Remote Desktop Users
  • Linux: root, sudo/wheel (administrátoři), users, www-data (webserver)

Autentizace a autorizace

Dva důležité pojmy, které se často pletou:

  • Autentizace (Authentication): „Kdo jsi?" – ověření identity uživatele
    • Heslo, PIN
    • Biometrie (otisk prstu, rozpoznání obličeje)
    • 2FA (dvoufaktorová autentizace) – heslo + SMS kód
  • Autorizace (Authorization): „Co smíš dělat?" – přidělení oprávnění po úspěšné autentizaci

💡 Příklad: Přihlásíš se heslem (autentizace). Systém ověří, zda máš právo číst daný soubor (autorizace).

Příkazy pro správu uživatelů

# WINDOWS (CMD) - spuštěno jako správce
net user                          # seznam uživatelů
net user jan                      # info o uživateli 'jan'
net user jan heslo123 /add        # vytvoří uživatele
net user jan /delete              # smaže uživatele
net localgroup administrators jan /add  # přidá do skupiny admin

# LINUX (jako root nebo sudo)
cat /etc/passwd                   # seznam uživatelů
useradd jan                       # vytvoří uživatele
useradd -m -s /bin/bash jan       # vytvoří s domovským adresářem
passwd jan                        # nastaví heslo
userdel jan                       # smaže uživatele
usermod -aG sudo jan              # přidá do skupiny sudo
groups jan                        # zobrazí skupiny uživatele

📊 Správa procesů

Co je proces?

Proces je běžící instance programu v paměti. Když spustíš Chrome, vytvoří se proces. Každý proces má:

  • PID (Process ID): Unikátní číslo procesu
  • Vlastní paměťový prostor: Procesy jsou od sebe izolované
  • Stav: Běžící, čekající, ukončený
  • Prioritu: Určuje, kolik času CPU dostane

Stavy procesu

Proces během svého života prochází různými stavy:

    [New] ──→ [Ready] ←──────→ [Running] ──→ [Terminated]
                  ↑                 │
                  │                 ↓
                  └───── [Waiting/Blocked]
  • New (Nový): Proces je právě vytvářen
  • Ready (Připraven): Čeká ve frontě na přidělení CPU
  • Running (Běžící): CPU právě vykonává jeho instrukce
  • Waiting/Blocked (Čekající): Čeká na I/O operaci nebo událost (např. vstup z klávesnice)
  • Terminated (Ukončen): Dokončil vykonávání nebo byl násilně ukončen

Plánovač procesů (Scheduler)

Součást OS, která rozhoduje, který proces dostane CPU a na jak dlouho. Existuje několik algoritmů:

  • FCFS (First Come First Served): Kdo přijde první, je obsloužen první. Jednoduchý, ale může způsobit „convoy effect" – krátké procesy čekají za dlouhým.
  • SJF (Shortest Job First): Nejkratší úloha jde první. Optimální průměrná doba čekání, ale těžko předvídat délku.
  • Round Robin: Každý proces dostane časové kvantum (např. 100 ms), pak jde na konec fronty. Spravedlivé pro interaktivní systémy.
  • Priority Scheduling: Procesy mají priority, vyšší priorita = dříve obsloužen. Riziko „starvation" pro nízké priority.

Vlákna (Threads)

Vlákno je lehčí jednotka provádění v rámci procesu. Jeden proces může mít více vláken, která:

  • Sdílejí paměťový prostor procesu (proměnné, data)
  • Mají vlastní zásobník a registry
  • Přepínání mezi vlákny je rychlejší než mezi procesy

💡 Příklad: Webový prohlížeč – jedno vlákno stahuje obrázky, druhé vykresluje stránku, třetí přehrává video.

Příkazy pro správu procesů

# WINDOWS
tasklist                    # seznam procesů
tasklist | find "chrome"    # hledání procesu
taskkill /PID 1234          # ukončení podle PID
taskkill /IM chrome.exe /F  # ukončení podle jména (F = force)
Ctrl+Shift+Esc              # Správce úloh (GUI)

# LINUX
ps aux                      # všechny procesy (detailně)
ps aux | grep firefox       # hledání procesu
top                         # živý přehled (stiskni q pro ukončení)
htop                        # vylepšený top (interaktivní)
kill 1234                   # ukončení (SIGTERM - žádost)
kill -9 1234                # násilné ukončení (SIGKILL)
killall firefox             # ukončí všechny procesy s daným jménem
nice -n 10 ./program        # spuštění s nižší prioritou

⚙️ Správa služeb

Co je služba/démon?

Služba (Windows) nebo démon (Linux) je proces běžící na pozadí bez uživatelského rozhraní. Poskytuje funkcionalitu pro systém nebo síť.

💡 Příklady služeb:

  • Webový server (Apache, Nginx, IIS)
  • Databázový server (MySQL, PostgreSQL)
  • DHCP, DNS klient
  • Antivirový monitoring
  • Tisk (Print Spooler)

Typy spouštění služeb

  • Automaticky: Spustí se při startu systému (např. antivirus)
  • Automaticky (zpožděně): Po startu systému s malým zpožděním
  • Manuálně: Spustí se jen na vyžádání (např. Bluetooth)
  • Zakázáno: Služba se nespustí, dokud nezměníš nastavení

Příkazy pro správu služeb

# WINDOWS
services.msc                    # GUI správce služeb
sc query                        # seznam služeb
sc query "Windows Update"       # info o konkrétní službě
sc start "Windows Update"       # spuštění služby
sc stop "Windows Update"        # zastavení
net start "Print Spooler"       # alternativa pro start
net stop "Print Spooler"        # alternativa pro stop

# LINUX (systemd - moderní distribuce)
systemctl list-units --type=service     # seznam služeb
systemctl status apache2                # stav služby
systemctl start apache2                 # spuštění
systemctl stop apache2                  # zastavení
systemctl restart apache2               # restart
systemctl enable apache2                # povolení autostartu
systemctl disable apache2               # zakázání autostartu

# Logy služby
journalctl -u apache2                   # log konkrétní služby

🎓 Shrnutí pro maturitu

  • Uživatelé: Admin má plná práva, standardní uživatel omezená
  • Autentizace = kdo jsi, autorizace = co můžeš
  • Proces = běžící program, má PID a vlastní paměť
  • Stavy procesu: New → Ready → Running → Waiting → Terminated
  • Vlákna sdílejí paměť procesu, rychlejší přepínání
  • Služba/démon = proces na pozadí bez UI
  • Windows: tasklist, taskkill, sc, services.msc
  • Linux: ps, top, kill, systemctl
