Téma 11 - Programování

📖 Co je dědičnost?

Dědičnost je mechanismus OOP, který umožňuje vytvořit novou třídu na základě existující. Potomek (derived class) přebírá vlastnosti a metody předka (base class) a může je rozšířit nebo upravit.

💡 Analogie:

  • Předek: Zvíře (má vlastnosti: jméno, věk; metody: jíst, spát)
  • Potomek: Pes (dědí vše od Zvířete + má navíc: štěkat, aportovat)

Výhody dědičnosti:

  • Znovupoužití kódu: Nepíšeš stejný kód dvakrát
  • Hierarchie: Logická organizace tříd
  • Polymorfismus: Umožňuje pracovat s objekty obecně

Základní syntaxe dědičnosti

// Bázová (rodičovská) třída
class Zvire {
protected:              // Přístupné v této třídě i v potomcích
    string jmeno;
    int vek;
    
public:
    Zvire(string j, int v) : jmeno(j), vek(v) {
        cout << "Zvíře " << jmeno << " vytvořeno" << endl;
    }
    
    void predstavSe() {
        cout << "Jsem " << jmeno << ", " << vek << " let." << endl;
    }
};

// Odvozená (potomek) třída
class Pes : public Zvire {    // Dědí od Zvire
public:
    string plemeno;
    
    // Konstruktor volá konstruktor předka
    Pes(string j, int v, string p) : Zvire(j, v), plemeno(p) {
        cout << "Pes plemene " << plemeno << " vytvořen" << endl;
    }
    
    // Nová metoda pouze pro Psa
    void stekej() {
        cout << jmeno << " říká: Haf haf!" << endl;
    }
};

// Použití
Pes rex("Rex", 3, "Německý ovčák");
rex.predstavSe();    // Zděděná metoda: "Jsem Rex, 3 let."
rex.stekej();        // Vlastní metoda: "Rex říká: Haf haf!"

Úrovně přístupu při dědění

Klíčové slovo za dvojtečkou určuje, jak se změní viditelnost zděděných členů:

class Potomek : public Predek    // Nejčastější
class Potomek : protected Predek
class Potomek : private Predek   // Méně časté
Dědění public členy protected členy private členy
public → public → protected ❌ nedědí se
protected → protected → protected ❌ nedědí se
private → private → private ❌ nedědí se

💡 V 99% případů používej public dědičnost!

Virtuální metody a polymorfismus

Klíčové slovo virtual umožňuje, aby potomek přepsal chování metody předka. Tak funguje polymorfismus.

class Zvire {
public:
    // BEZ virtual - volá se metoda podle TYPU ukazatele
    void zvuk() {
        cout << "Nějaký zvuk" << endl;
    }
    
    // S virtual - volá se metoda podle SKUTEČNÉHO OBJEKTU
    virtual void zvukVirtual() {
        cout << "Nějaký zvuk" << endl;
    }
};

class Pes : public Zvire {
public:
    void zvuk() {                    // Přepsání (nevirtuální)
        cout << "Haf!" << endl;
    }
    
    void zvukVirtual() override {    // Přepsání virtuální metody
        cout << "Haf!" << endl;      // override je volitelné, ale doporučené
    }
};

class Kocka : public Zvire {
public:
    void zvukVirtual() override {
        cout << "Mňau!" << endl;
    }
};

// Demonstrace polymorfismu
Zvire* zvirata[3];
zvirata[0] = new Zvire();
zvirata[1] = new Pes();
zvirata[2] = new Kocka();

// Nevirtuální metoda - vždy volá Zvire::zvuk()
for (int i = 0; i < 3; i++) {
    zvirata[i]->zvuk();    // 3x "Nějaký zvuk" 😕
}

// Virtuální metoda - volá správnou verzi podle objektu
for (int i = 0; i < 3; i++) {
    zvirata[i]->zvukVirtual();  // "Nějaký zvuk", "Haf!", "Mňau!" 🎉
}

⚠️ Klíčové: Bez virtual se volá metoda podle typu ukazatele. S virtual se volá podle skutečného typu objektu.

Čistě virtuální metody (abstraktní třídy)

Metoda bez implementace, kterou MUSÍ potomek přepsat. Dělá z třídy abstraktní třídu (nelze vytvořit instanci).

class Tvar {
public:
    // Čistě virtuální metoda - nemá tělo, končí = 0
    virtual double obsah() = 0;
    virtual double obvod() = 0;
    
    // Virtuální destruktor (důležité pro správné uvolnění paměti)
    virtual ~Tvar() { }
};

// Tvar tvar;  // CHYBA! Abstraktní třídu nelze instancovat

class Obdelnik : public Tvar {
    double sirka, vyska;
public:
    Obdelnik(double s, double v) : sirka(s), vyska(v) { }
    
    double obsah() override {    // MUSÍ implementovat
        return sirka * vyska;
    }
    
    double obvod() override {    // MUSÍ implementovat
        return 2 * (sirka + vyska);
    }
};

class Kruh : public Tvar {
    double polomer;
public:
    Kruh(double r) : polomer(r) { }
    
    double obsah() override {
        return 3.14159 * polomer * polomer;
    }
    
    double obvod() override {
        return 2 * 3.14159 * polomer;
    }
};

// Polymorfní použití
Tvar* tvary[2] = { new Obdelnik(5, 3), new Kruh(2) };
for (auto t : tvary) {
    cout << "Obsah: " << t->obsah() << endl;
}

Vícenásobná dědičnost

C++ umožňuje dědit z více tříd najednou. Může způsobit komplikace (diamond problem).

class Letajici {
public:
    void let() { cout << "Letím!" << endl; }
};

class Plavajici {
public:
    void plav() { cout << "Plavu!" << endl; }
};

// Kachna dědí z obou
class Kachna : public Letajici, public Plavajici {
public:
    void kvak() { cout << "Kvak!" << endl; }
};

Kachna donald;
donald.let();    // Z Letajici
donald.plav();   // Z Plavajici
donald.kvak();   // Vlastní

⚠️ Pozor na diamond problem: Když dvě bázové třídy dědí ze stejného předka. Řeší se virtuální dědičností (virtual public).

Volání konstruktorů předka

class Predek {
protected:
    int hodnota;
public:
    Predek(int h) : hodnota(h) {
        cout << "Konstruktor Předka" << endl;
    }
};

class Potomek : public Predek {
    string text;
public:
    // Voláme konstruktor předka v inicializačním seznamu
    Potomek(int h, string t) : Predek(h), text(t) {
        cout << "Konstruktor Potomka" << endl;
    }
};

Potomek p(42, "Ahoj");
// Výstup:
// Konstruktor Předka
// Konstruktor Potomka

🎓 Shrnutí pro maturitu

  • Dědičnost: class Potomek : public Predek
  • protected: Přístupné v třídě i potomcích, ne zvenčí
  • virtual: Umožňuje přepsání metody v potomkovi (polymorfismus)
  • override: Označuje, že přepisujeme virtuální metodu
  • = 0: Čistě virtuální metoda → abstraktní třída
  • Polymorfismus: Ukazatel na předka může ukazovat na potomka
  • Konstruktor potomka volá konstruktor předka v inicializačním seznamu
