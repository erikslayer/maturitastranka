💻 Rozdělení programovacích jazyků

Téma 1 - Programování

💻 Programování 📚 Maturitní okruh

📖 Úvod do programovacích jazyků

Programovací jazyk je formální jazyk, který slouží k zápisu algoritmů a instrukcí pro počítač. Existuje více než 700 programovacích jazyků, ale v praxi se běžně používá jen několik desítek. Každý jazyk má své specifické vlastnosti, výhody a nevýhody, které ho činí vhodným pro různé typy úloh.

Pochopení rozdílů mezi jazyky je důležité pro správný výběr nástroje pro daný projekt – stejně jako bychom nepoužili kladivo na šroubování, nepoužijeme ani Python na psaní ovladačů hardwaru.

Rozdělení podle úrovně abstrakce

Úroveň abstrakce určuje, jak blízko je jazyk k strojovému kódu (nízká úroveň) nebo k lidskému uvažování (vysoká úroveň).

Nízkoúrovňové jazyky

Tyto jazyky jsou velmi blízko hardwaru a poskytují přímou kontrolu nad pamětí a procesorem. Jsou rychlé, ale složité na psaní.

  • Strojový kód: Binární instrukce procesoru (0 a 1) – přímo spustitelné CPU, ale prakticky nečitelné pro člověka
  • Assembler: Mnemotechnické značky pro instrukce (MOV, ADD, JMP) – každá instrukce odpovídá jedné strojové instrukci

💡 Příklad assembleru (x86):

MOV AX, 5      ; ulož hodnotu 5 do registru AX
ADD AX, 3      ; přičti 3 k registru AX
; Výsledek: AX = 8

🎯 Použití: Ovladače hardwaru, operační systémy, embedded systémy, optimalizace kritických částí kódu.

Vysokoúrovňové jazyky

Jsou bližší lidskému jazyku a abstrahují od hardwaru. Jeden příkaz může znamenat desítky strojových instrukcí.

  • Snadnější čtení a psaní kódu
  • Přenositelnost mezi platformami
  • Příklady: C, C++, Java, Python, C#, JavaScript, PHP

💡 Stejná operace ve vysokoúrovňovém jazyce:

int vysledek = 5 + 3;  // Výsledek: 8

Všimni si, jak je vysokoúrovňový zápis kratší a čitelnější!

Rozdělení podle způsobu překladu

Počítač rozumí pouze strojovému kódu. Zdrojový kód (to, co programátor napíše) musí být nějak převeden na instrukce procesoru. To lze udělat třemi způsoby:

Kompilované jazyky

Kompilátor přeloží celý zdrojový kód najednou do spustitelného souboru (.exe) ještě před spuštěním programu.

  • Proces: Zdrojový kód → Kompilátor → Strojový kód (.exe)
  • Výhody: Rychlé spuštění, optimalizace kódu, chyby odhaleny při kompilaci
  • Nevýhody: Nutnost rekompilace po změnách, závislost na platformě
  • Příklady: C, C++, Rust, Go

Interpretované jazyky

Interpret čte a vykonává kód řádek po řádku za běhu programu.

  • Proces: Zdrojový kód → Interpret → Okamžité vykonání
  • Výhody: Rychlý vývoj, snadné ladění, přenositelnost
  • Nevýhody: Pomalejší běh (překlad za běhu), nutnost mít interpret nainstalovaný
  • Příklady: Python, JavaScript, PHP, Ruby

Jazyky s mezikódem (hybridní)

Kombinace obou přístupů. Kód se nejprve zkompiluje do mezikódu (bytecode), který pak vykonává virtuální stroj.

  • Proces: Zdrojový kód → Kompilátor → Bytecode → Virtuální stroj
  • Výhody: „Napiš jednou, spusť kdekoliv" – nezávislost na platformě
  • Příklady:
    • Java → kompiluje do bytecode → běží na JVM (Java Virtual Machine)
    • C# → kompiluje do IL → běží na .NET CLR (Common Language Runtime)

⚠️ Častá chyba: Studenti si pletou kompilaci a interpretaci. Klíčový rozdíl je kdy se překlad děje – kompilace před spuštěním, interpretace za běhu.

Programovací paradigmata

Paradigma je způsob přemýšlení o tom, jak strukturovat program. Většina moderních jazyků podporuje více paradigmat.

Procedurální programování

Program je sekvence příkazů, které se vykonávají postupně. Kód je organizován do procedur (funkcí).

  • Top-down přístup – řešíme problém shora dolů
  • Příklady: C, Pascal, starší BASIC
// Procedurální přístup v C
void pozdrav() {
    printf("Ahoj!");
}
int main() {
    pozdrav();  // volání procedury
    return 0;
}

Objektově orientované programování (OOP)

Program je tvořen objekty, které mají vlastnosti (data) a metody (chování). Základní pilíře:

  • Zapouzdření: Data a metody jsou spojeny v objektu, vnitřní implementace je skryta
  • Dědičnost: Třídy mohou dědit vlastnosti od jiných tříd
  • Polymorfismus: Stejná metoda se může chovat různě pro různé objekty
  • Příklady: Java, C++, C#, Python
// OOP přístup v C#
class Pes {
    public string Jmeno { get; set; }
    public void Stekej() {
        Console.WriteLine("Haf!");
    }
}
Pes rex = new Pes();
rex.Jmeno = "Rex";
rex.Stekej();  // "Haf!"

Funkcionální programování

Důraz na čisté funkce bez vedlejších efektů. Data jsou neměnná (immutabilní).

  • Funkce jako první hodnoty – lze je předávat jako argumenty
  • Vhodné pro paralelní zpracování
  • Příklady: Haskell, F#, Scala, Erlang

Základní pojmy programování

Tyto pojmy jsou základem, které musíš znát:

  • Algoritmus: Konečná posloupnost kroků vedoucí k vyřešení problému. Musí být jednoznačný, konečný a efektivní.
  • Program: Implementace (zápis) algoritmu v konkrétním programovacím jazyce.
  • Syntaxe: Pravidla pro správný zápis kódu – jak musí kód vypadat, abychom ho mohli přeložit.
  • Sémantika: Význam programových konstrukcí – co kód ve skutečnosti dělá.
  • IDE (Integrated Development Environment): Vývojové prostředí s editorem, kompilátorem a debuggerem v jednom (Visual Studio, VS Code, IntelliJ IDEA).
  • Debugger: Nástroj pro ladění programů – umožňuje krokovat kód a sledovat hodnoty proměnných.
  • Kompilace: Překlad zdrojového kódu do strojového kódu.
  • Runtime: Běhové prostředí, ve kterém se program vykonává.

🎓 Shrnutí pro maturitu

  • Nízkoúrovňové jazyky = blízko hardwaru (assembler), vysokoúrovňové = blízko člověku (Python, Java)
  • Kompilované = překlad před spuštěním (C++), interpretované = překlad za běhu (Python)
  • Paradigmata: procedurální (funkce), OOP (objekty), funkcionální (čisté funkce)
  • Umět vysvětlit rozdíl mezi syntaxí (jak píšu) a sémantikou (co to dělá)
