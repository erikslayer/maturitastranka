📖 Co je SQL?

SQL (Structured Query Language) je standardní jazyk pro práci s relačními databázemi. Umožňuje data vybírat, vkládat, měnit a mazat. SQL příkazy se dělí na:

  • DQL (Data Query Language): SELECT – výběr dat
  • DML (Data Manipulation Language): INSERT, UPDATE, DELETE – manipulace s daty
  • DDL (Data Definition Language): CREATE, ALTER, DROP – struktura databáze

💡 Pro příklady použijeme tabulku:

+----+---------------+-------------------+------+----------+
| id | jmeno         | email             | vek  | mesto    |
+----+---------------+-------------------+------+----------+
| 1  | Jan Novák     | jan@email.cz      | 25   | Praha    |
| 2  | Eva Svobodová | eva@email.cz      | 30   | Brno     |
| 3  | Petr Černý    | petr@email.cz     | 22   | Praha    |
| 4  | Marie Bílá    | NULL              | 45   | Ostrava  |
+----+---------------+-------------------+------+----------+

SELECT – Výběr dat

Nejpoužívanější příkaz SQL. Vybírá data z tabulky podle zadaných kritérií.

Základní syntaxe

-- Všechny sloupce a všechny řádky
SELECT * FROM uzivatele;

-- Jen vybrané sloupce
SELECT jmeno, email FROM uzivatele;

-- Výsledek:
-- +---------------+-------------------+
-- | jmeno         | email             |
-- +---------------+-------------------+
-- | Jan Novák     | jan@email.cz      |
-- | Eva Svobodová | eva@email.cz      |
-- | ...           | ...               |

Alias (přejmenování sloupce)

SELECT jmeno AS 'Jméno uživatele', vek AS 'Věk' FROM uzivatele;

-- Výsledek má přejmenované záhlaví sloupců

DISTINCT – unikátní hodnoty

-- Jaká města máme v databázi? (bez duplicit)
SELECT DISTINCT mesto FROM uzivatele;

-- Výsledek: Praha, Brno, Ostrava (každé jen jednou)

WHERE – Filtrování

Klauzule WHERE omezuje výsledky na řádky splňující podmínku. Bez WHERE dostaneme všechny řádky.

-- Uživatelé starší 18 let
SELECT * FROM uzivatele WHERE vek > 18;

-- Uživatelé z Prahy
SELECT * FROM uzivatele WHERE mesto = 'Praha';
-- Výsledek: Jan Novák a Petr Černý

-- Rozsah hodnot
SELECT * FROM uzivatele WHERE vek BETWEEN 18 AND 30;
-- BETWEEN je inkluzivní (včetně 18 i 30)

-- Vyhledávání v textu (LIKE)
SELECT * FROM uzivatele WHERE jmeno LIKE 'J%';
-- % = libovolný počet znaků → najde "Jan Novák"

SELECT * FROM uzivatele WHERE jmeno LIKE '%ová';
-- Najde všechny končící na "ová" → "Eva Svobodová"

-- Více možností (IN)
SELECT * FROM uzivatele WHERE mesto IN ('Praha', 'Brno');

-- Kontrola NULL hodnot
SELECT * FROM uzivatele WHERE email IS NULL;
-- Najde Marie Bílá (nemá email)

SELECT * FROM uzivatele WHERE email IS NOT NULL;
-- Najde všechny s emailem

⚠️ Pozor: Pro NULL nepoužíváme = NULL, ale IS NULL!

Logické operátory

-- AND – obě podmínky musí platit
SELECT * FROM uzivatele WHERE vek > 18 AND mesto = 'Praha';

-- OR – alespoň jedna podmínka
SELECT * FROM uzivatele WHERE vek < 18 OR vek > 65;

-- NOT – negace
SELECT * FROM uzivatele WHERE NOT mesto = 'Praha';
-- Nebo: WHERE mesto != 'Praha' (ekvivalent)

ORDER BY – Řazení

Určuje pořadí výsledků. Výchozí je vzestupně (ASC).

-- Vzestupně podle jména (A-Z)
SELECT * FROM uzivatele ORDER BY jmeno;
-- Nebo explicitně: ORDER BY jmeno ASC;

-- Sestupně podle věku (od nejstarších)
SELECT * FROM uzivatele ORDER BY vek DESC;

-- Kombinované řazení
SELECT * FROM uzivatele ORDER BY mesto, jmeno;
-- Nejprve seřadí podle města, pak v každém městě podle jména

LIMIT – Omezení výsledků

-- Prvních 10 záznamů
SELECT * FROM uzivatele LIMIT 10;

-- 10 záznamů od pozice 5 (pro stránkování)
SELECT * FROM uzivatele LIMIT 5, 10;
-- Přeskočí prvních 5, vrátí dalších 10

-- Praktický příklad: Top 3 nejstarší
SELECT * FROM uzivatele ORDER BY vek DESC LIMIT 3;

GROUP BY a HAVING

GROUP BY seskupí řádky podle hodnoty a umožní použít agregační funkce (COUNT, SUM, AVG...).

HAVING filtruje skupiny (jako WHERE, ale po seskupení).

-- Kolik uživatelů je v každém městě?
SELECT mesto, COUNT(*) AS pocet 
FROM uzivatele 
GROUP BY mesto;

-- Výsledek:
-- +----------+-------+
-- | mesto    | pocet |
-- +----------+-------+
-- | Praha    | 2     |
-- | Brno     | 1     |
-- | Ostrava  | 1     |
-- +----------+-------+

-- Jen města s více než 1 uživatelem (HAVING)
SELECT mesto, COUNT(*) AS pocet 
FROM uzivatele 
GROUP BY mesto 
HAVING COUNT(*) > 1;

-- Výsledek: jen Praha (2)

💡 Rozdíl WHERE vs HAVING:

  • WHERE: Filtruje před seskupením (jednotlivé řádky)
  • HAVING: Filtruje po seskupení (celé skupiny)

INSERT – Vkládání dat

Přidává nové záznamy do tabulky.

-- Vložení jednoho záznamu (všechny sloupce)
INSERT INTO uzivatele (jmeno, email, vek, mesto) 
VALUES ('Jan Novák', 'jan@email.cz', 25, 'Praha');

-- Vložení více záznamů najednou
INSERT INTO uzivatele (jmeno, email, vek, mesto) VALUES 
('Eva Svobodová', 'eva@email.cz', 30, 'Brno'),
('Petr Černý', 'petr@email.cz', 28, 'Praha');

-- Pokud má sloupec AUTO_INCREMENT (id), nemusíme ho uvádět

UPDATE – Aktualizace dat

Mění existující záznamy. Vždy používej WHERE, jinak změníš všechny řádky!

-- Změna věku konkrétního uživatele
UPDATE uzivatele SET vek = 26 WHERE id = 1;

-- Změna více hodnot
UPDATE uzivatele 
SET mesto = 'Praha', vek = vek + 1 
WHERE jmeno = 'Jan Novák';

-- ⚠️ POZOR - toto změní VŠECHNY záznamy!
UPDATE uzivatele SET mesto = 'Praha';
-- Bez WHERE = aktualizace celé tabulky!

DELETE – Mazání dat

Odstraňuje záznamy. Vždy používej WHERE!

-- Smazání konkrétního záznamu
DELETE FROM uzivatele WHERE id = 1;

-- Smazání podle podmínky
DELETE FROM uzivatele WHERE vek < 18;

-- ⚠️ POZOR - toto smaže VŠECHNY záznamy!
DELETE FROM uzivatele;
-- Bez WHERE = prázdná tabulka!

Pořadí klauzulí v SELECT

Klauzule musí být ve správném pořadí:

SELECT sloupce
FROM tabulka
WHERE podmínka
GROUP BY sloupce
HAVING podmínka_skupiny
ORDER BY sloupce
LIMIT pocet;

🎓 Shrnutí pro maturitu

  • SELECT: Výběr dat – SELECT sloupce FROM tabulka
  • WHERE: Filtrování – podmínky, LIKE, IN, BETWEEN, IS NULL
  • ORDER BY: Řazení – ASC (vzestupně) / DESC (sestupně)
  • GROUP BY + HAVING: Seskupení a filtrování skupin
  • INSERT: Vkládání – INSERT INTO ... VALUES
  • UPDATE: Změna – vždy s WHERE!
  • DELETE: Mazání – vždy s WHERE!
