⚡ Procesory (CPU)

Téma 3 - Hardware

🖥️ Hardware 📚 Maturitní okruh

Procesor (CPU - Central Processing Unit) je "mozek" počítače, který vykonává instrukce programů a zpracovává data.

1. Základní architektura CPU

Von Neumannova architektura

  • Program a data jsou uloženy ve společné paměti
  • Instrukce se zpracovávají sekvenčně
  • Komponenty: ALU, řadič, registry, sběrnice
  • Bottleneck: Von Neumannovo úzké hrdlo (sdílená sběrnice)

Harvardská architektura

  • Oddělená paměť pro program a data
  • Možnost současného přístupu k oběma pamětem
  • Používá se v DSP a mikrokontrolérech

2. Komponenty procesoru

ALU (Arithmetic Logic Unit)

  • Provádí aritmetické operace (sčítání, násobení...)
  • Provádí logické operace (AND, OR, XOR, NOT)
  • Moderní CPU mají více ALU jednotek

Řadič (Control Unit)

  • Dekóduje instrukce
  • Řídí tok dat mezi komponenty
  • Generuje řídící signály

Registry

  • Nejrychlejší paměťové buňky přímo v CPU
  • Obecné registry: Pro dočasná data (EAX, EBX v x86)
  • Programový čítač (PC): Adresa další instrukce
  • Stavový registr: Příznaky (zero, carry, overflow)

3. Instrukční cyklus

Základní cyklus zpracování instrukce (Fetch-Decode-Execute):

  1. Fetch: Načtení instrukce z paměti
  2. Decode: Dekódování instrukce (co má dělat)
  3. Execute: Provedení operace
  4. Write-back: Zápis výsledku

Pipeline (zřetězení)

  • Překrývání fází více instrukcí současně
  • Zvyšuje propustnost (throughput)
  • Hazardy: Datové, řídící, strukturální konflikty
  • Branch prediction: Predikce větvení pro snížení penalizace

4. Jádra a vlákna

Jádra (Cores)

  • Fyzické výpočetní jednotky v CPU
  • Každé jádro může zpracovávat instrukce nezávisle
  • Více jader = lepší multitasking a paralelní zpracování

Vlákna (Threads)

  • SMT (Simultaneous Multithreading): Intel = Hyper-Threading
  • Jedno jádro může zpracovávat 2 vlákna současně
  • Využívá nevyužité zdroje jádra
  • Příklad: 8 jader / 16 vláken

Hybridní architektura (Intel 12+ gen)

  • P-jádra (Performance): Výkonná jádra pro náročné úlohy
  • E-jádra (Efficient): Úsporná jádra pro pozadí
  • Lepší poměr výkon/spotřeba

5. Cache paměť

Rychlá mezipaměť mezi CPU a RAM:

  • L1 Cache: Nejrychlejší, nejmenší (32-64 KB na jádro), rozdělena na instrukce a data
  • L2 Cache: Větší, pomalejší (256 KB - 1 MB na jádro)
  • L3 Cache: Sdílená mezi jádry (8-64+ MB), nejpomalejší cache

Principy fungování cache

  • Cache hit: Data nalezena v cache (rychlé)
  • Cache miss: Data musí být načtena z RAM (pomalé)
  • Hit rate: Procento úspěšných přístupů

6. Taktovací frekvence

  • Base clock: Základní frekvence (např. 3.6 GHz)
  • Boost clock: Maximální frekvence při zátěži (např. 5.2 GHz)
  • Přetaktování (OC): Zvýšení frekvence nad specifikaci
  • Vyšší frekvence = vyšší výkon, ale i spotřeba a teplota

7. Výrobní proces

  • Měří se v nanometrech (nm)
  • Menší proces: Více tranzistorů, nižší spotřeba, nižší teplota
  • Aktuální procesy: 7nm, 5nm, 4nm, 3nm
  • Výrobci: TSMC, Samsung, Intel Foundry

8. Instrukční sady

CISC (Complex Instruction Set Computer)

  • Mnoho komplexních instrukcí
  • Příklad: x86, x86-64 (Intel, AMD)
  • Dominuje v desktopech a serverech

RISC (Reduced Instruction Set Computer)

  • Jednoduché, optimalizované instrukce
  • Příklad: ARM (mobily, Apple M-series), RISC-V
  • Vyšší energetická efektivita

Rozšíření instrukční sady

  • SSE, AVX: Vektorové operace (SIMD)
  • AES-NI: Hardwarové šifrování
  • AVX-512: Pokročilé vektorové operace

9. Hlavní výrobci

Intel

  • Core i3, i5, i7, i9 (desktop)
  • Xeon (servery)
  • Aktuální: 14. generace (Raptor Lake Refresh)

AMD

  • Ryzen 3, 5, 7, 9 (desktop)
  • EPYC (servery)
  • Aktuální: Ryzen 9000 (Zen 5)

Apple

  • M1, M2, M3, M4 (ARM-based)
  • Vysoká efektivita, integrovaná GPU

10. TDP a spotřeba

  • TDP (Thermal Design Power): Tepelný výkon, který musí chladič odvést
  • Typické hodnoty: 65W (běžné), 125W (výkonné), 200W+ (HEDT)
  • Skutečná spotřeba může být vyšší při boost

🎓 Shrnutí pro maturitu

  • CPU = mozek počítače, vykonává instrukce
  • Komponenty: ALU, řadič, registry, cache
  • Instrukční cyklus: Fetch → Decode → Execute → Write-back
  • Jádra = fyzické jednotky, vlákna = logické (HT/SMT)
  • Cache: L1 (nejrychlejší) > L2 > L3 (největší)
  • CISC (x86) vs RISC (ARM)
  • Frekvence: Base clock a Boost clock (GHz)
  • Výrobci: Intel (Core), AMD (Ryzen), Apple (M-series)
